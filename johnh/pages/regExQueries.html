Regular Expression Queries Explained

				db.dogs.find(
					{age:{$lt: 20}}, < -- QUERY
					{"_id": 0, "name": 1, age: 1} <-- PROJECTION
				)

Queries in mongo are composed of two main components. The actual query is what you want to find. The projection filters what information is shown to you. 

					{age:{$lt: 20}} < -- QUERY

The query illustrated above is a simple version. Essentially, it looks for any record that has a field, 'age', that is less than 20.

				{$or:[{"Pokemon":null},{"Type":"Ice"}]}
				{$and:[{tricks:"sit"},{tricks:"roll over"}]}

This example shows a more complicated query utilizing or and and statements. The first example will return a record if either the pokemon is null or the
type is ice. On the other hand, the next example will return the record only if the record contains both the tricks sit and roll over.

				{name: /T.*$/}

This example shows regular expressions. The slashes (/) represent a start of the reg ex sequence. This particular search will return records that contain
a T followed by any character. The dollar sign represents the end of a word. So essentially it looks for any word that contains a T in the name field.
